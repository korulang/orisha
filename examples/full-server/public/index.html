How this works: This page is a static file embedded in the binary at compile time. The POST /api/eval endpoint runs a budgeted Koru interpreter against compiled event handlers. The GET /api/health endpoint returns a static JSON response. All three modes — static files, dynamic routes, and live interpretation — compose through a single orisha:handler using |> continuations.
